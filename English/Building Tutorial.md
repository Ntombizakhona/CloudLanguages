### **Step Three: The CLEAR Framework for PartyRock Prompts**

Use this framework for every prompt:

**C**ontext: What is the app‚Äôs purpose?

**L**ayout: How should it look and function?

**E**xpected inputs: What data will users provide?

**A**ctions: What should the app do with that data?

**R**esponse format: How should results be presented?

### **Step Five: Advanced Prompt Techniques**

**Use Specific Examples**

**Instead of:** _Make it user-friendly_

**Try:** Include clear labels, helpful tooltips, and error messages like **_‚ÄòPlease enter a valid email address‚Äô_** when validation fails

**Define the Persona:** Respond as an experienced DevOps engineer who explains complex concepts in simple terms, uses real-world examples, and always considers security implications.

**Set Constraints:** Keep responses under 200 words, use bullet points for lists, and always include at least one actionable next step.

**Chain Prompts for Complex Apps**

1. _First prompt:_ Define the app structure and main features

2. _Second prompt:_ Refine the user interface and user experience

3. _Third prompt:_ Add advanced features and error handling

### **Step Six: Testing and Iteration**

**The 3-Test Rule:**

1. Test with expected inputs: _happy path_

2. Test with edge cases: _empty inputs, very long text_

3. Test with unexpected inputs: _wrong data types_

### Step Seven: **Refinement Process**
